# @steflsd/ficta-web

Internal chat UI for the ficta PII/secret redaction gateway — a [TanStack Start](https://tanstack.com/start)
app using [TanStack AI](https://github.com/tanstack/ai) for any-model / bring-your-own-key chat.

Every model call is routed **through the ficta proxy** (`src/lib/model-adapter.ts` sets each adapter's
`baseURL` to the proxy), so registered secrets and best-effort detected PII are tokenized before the
vendor and restored on the way back. Users see restored values; the vendor sees surrogates for the
values ficta actually registered or detected. PII detection is a reduction layer, not a completeness
guarantee.

```
browser (useChat)
  → /api/chat        server route (src/routes/api/chat.ts)
  → ficta proxy      redact → forward → restore   (FICTA_PROXY_URL)
  → OpenAI / Anthropic
```

Current app capabilities:

- server-side BYO OpenAI/Anthropic keys; keys are never sent to the browser;
- chat history and settings backed by embedded PGlite by default, or Postgres via `DATABASE_URL`;
- ChatGPT-style sidebar with autosaving settings for default model, instance name, and model allow-list;
- protection badge/banner polling the proxy's safe `/__ficta/status` endpoint, including Presidio
  fail-open vs fail-closed outage posture;
- text-file attachments inlined into chat requests so ficta can redact them; PDF/DOCX uploads are
  blocked with a prompt to paste relevant context until local extraction exists;
- optional WorkOS AuthKit organizations/workspaces (`AUTH_PROVIDER=workos`), with open local
  self-hosting via `AUTH_PROVIDER=none` by default.

## Run

> Status: installs from the repo root, typechecks, and builds (client + SSR). `src/routeTree.gen.ts`
> is generated by the Vite plugin on first `dev`/`build` (git-ignored). The repo has local proxy
> loopback coverage; live vendor checks are manual by design because they send test content to a real
> provider. Use fake PII for those checks.

1. From the repo root: `pnpm install` (installs this app's deps too — it's a workspace member).
2. **Provide config + model API keys.** Values are read server-side only (never sent to the browser)
   and can come from Doppler *or* a `.env` file. The server needs `OPENAI_API_KEY` and/or
   `ANTHROPIC_API_KEY`; the UI reads `FICTA_PROXY_URL`, and the proxy reads `FICTA_PII_ENABLED`.

   `pnpm dev` is the single repo-root entrypoint. It runs `scripts/dev.mjs`, which uses
   `doppler run -- pnpm dev:all` when Doppler project/config metadata is detected, otherwise loads
   local `.env` files and runs `pnpm dev:all` without Doppler.

   ```sh
   pnpm dev                 # auto: Doppler when configured, otherwise local .env
   pnpm dev -- --doppler    # force Doppler
   pnpm dev -- --no-doppler # force local .env
   ```

   For local `.env` fallback, copy the example and fill it in:

   ```sh
   cp apps/web/.env.example apps/web/.env
   # then edit apps/web/.env and set OPENAI_API_KEY and/or ANTHROPIC_API_KEY
   pnpm dev
   ```

   Then open http://localhost:4747. (`pnpm dev:proxy` is the proxy only; `pnpm web:dev` is the UI only.)

   This dev flow is the **web-UI** path. The coding agents don't need it — `ficta claude|codex|pi`
   starts its own ephemeral proxy per launch (see
   [`packages/ficta/docs/install.md`](../../packages/ficta/docs/install.md)).

**Precedence:** shell-exported and Doppler-injected values win. `scripts/dev.mjs` loads local `.env`
files only as fallback before launching the proxy + UI; `vite.config.ts` keeps loading
`apps/web/.env` as the same fallback when `pnpm web:dev` is run directly.

**PII redaction** is controlled by the **proxy** process (not the UI). Its durable home is the
proxy's own config: `ficta setup` prompts for `[pii] enabled` and defaults it on for the
web/standalone proxy (see
[`packages/ficta/docs/plugins.md#built-in-detector-plugin-pii`](../../packages/ficta/docs/plugins.md#built-in-detector-plugin-pii)).
Note the two defaults — an *unconfigured* proxy is off, but running the wizard turns it on. For a
one-off instead, pass it inline: `FICTA_PII_ENABLED=1 pnpm dev`; `FICTA_PII_ENABLED=0` forces it off.

Backend selection is also proxy-side: `FICTA_PII_BACKEND=regex` is the built-in structured detector;
`FICTA_PII_BACKEND=presidio` is a first-class supported backend that calls a running Microsoft
Presidio analyzer sidecar. ficta does not launch that service — run it yourself, usually with Docker
for local development:

```sh
docker run --rm -p 5002:3000 mcr.microsoft.com/presidio-analyzer:latest
FICTA_PII_ENABLED=1 \
FICTA_PII_BACKEND=presidio \
FICTA_PII_PRESIDIO_URL=http://127.0.0.1:5002 \
pnpm dev
```

If the selected backend is unavailable, ficta either skips detection and forwards (fail-open) or
blocks before the model (fail-closed), according to `[detection] fail_closed` / `[pii] fail_closed`.
The web UI polls `/__ficta/status` through a server function and surfaces that posture in the
badge/banner.

> The ficta proxy also loads secrets to redact from the same Doppler context via its own Doppler
> registry plugin (`packages/ficta/src/plugins/doppler.ts`), so running under `doppler run` feeds both
> the web app's env and the proxy's redaction registry.

## Configuration (server-side only)

Web app env:

| Env var | Purpose | Default |
| --- | --- | --- |
| `FICTA_PROXY_URL` | ficta proxy base URL the model adapters point at | `http://127.0.0.1:8787` |
| `OPENAI_API_KEY` | OpenAI-compatible model key; passed through ficta to the vendor as an auth header | — |
| `ANTHROPIC_API_KEY` | Anthropic model key; passed through ficta to the vendor as an auth header | — |
| `AUTH_PROVIDER` | `none` for open local/self-hosted mode, or `workos` for AuthKit | `none` |
| `WORKOS_CLIENT_ID`, `WORKOS_API_KEY`, `WORKOS_REDIRECT_URI`, `WORKOS_COOKIE_PASSWORD` | WorkOS AuthKit + organization/workspace support | — |
| `DATABASE_URL` | Postgres connection for multi-process/serverless deployments | embedded PGlite when unset |
| `FICTA_WEB_DATA_DIR` | PGlite data directory when `DATABASE_URL` is unset | `.data/pglite` |

Proxy-side env commonly used with the web UI:

| Env var | Purpose | Default |
| --- | --- | --- |
| `FICTA_PII_ENABLED` | Enable best-effort PII detection for the standalone/web proxy | unconfigured proxy: `0`; `ficta setup`: prompted/default yes |
| `FICTA_PII_BACKEND` | Select `regex` or `presidio` PII backend | `regex` |
| `FICTA_PII_FAIL_CLOSED` | PII-specific outage policy: block instead of skip detection | `0` |
| `FICTA_PII_PRESIDIO_URL` | Presidio analyzer URL when using the `presidio` backend | local sidecar URL |

Model provider keys live only on the server (the `/api/chat` handler); they are never sent to the
browser. The proxy receives them only as auth headers that it intentionally passes through upstream.

## Seam

`createModelAdapter({ provider, model })` is the single place provider/model/key/base-URL are wired.
Swapping providers or pointing at a different gateway is a change here — nowhere else.
