# @steflsd/ficta-web

Internal chat UI for the ficta PII/secret redaction gateway — a [TanStack Start](https://tanstack.com/start)
app using [TanStack AI](https://github.com/tanstack/ai) for any-model / bring-your-own-key chat.

Every model call is routed **through the ficta proxy** (`src/lib/model-adapter.ts` sets each adapter's
`baseURL` to the proxy), so PII and secrets are tokenized before the vendor and restored on the way
back. The lawyer sees real values; the vendor never does.

```
browser (useChat)
  → /api/chat        server route (src/routes/api/chat.ts)
  → ficta proxy      redact → forward → restore   (FICTA_PROXY_URL)
  → OpenAI / Anthropic
```

## Run

> Status: installs from the repo root, typechecks, and builds (client + SSR). `src/routeTree.gen.ts`
> is generated by the Vite plugin on first `dev`/`build` (git-ignored). Not yet exercised end-to-end
> against a live proxy + vendor.

1. From the repo root: `pnpm install` (installs this app's deps too — it's a workspace member).
2. **Provide your model API keys** in `apps/web/.env` (git-ignored, loaded server-side only — never
   sent to the browser):

   ```sh
   cp apps/web/.env.example apps/web/.env
   # then edit apps/web/.env and set OPENAI_API_KEY and/or ANTHROPIC_API_KEY
   ```

3. Start **both** the proxy and the UI with one command from the repo root — turning on PII
   redaction at the proxy (that flag is read by the proxy process, so pass it in the shell):

   ```sh
   FICTA_PII_ENABLED=1 pnpm dev:all
   ```

   Then open http://localhost:3000. (`pnpm dev` alone is the proxy only; `pnpm web:dev` is the UI only.)

**Where each value goes:** `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` / `FICTA_PROXY_URL` → `apps/web/.env`
(the web server reads them). `FICTA_PII_ENABLED` → the **proxy** process (shell env, as above), since
the proxy — not the UI — does the redaction.

## Configuration (server-side only)

| Env var | Purpose | Default |
| --- | --- | --- |
| `FICTA_PROXY_URL` | ficta proxy base URL the model adapters point at | `http://127.0.0.1:8787` |
| `OPENAI_API_KEY` | firm key for OpenAI-compatible models (passes through ficta to the vendor) | — |
| `ANTHROPIC_API_KEY` | firm key for Anthropic models | — |

Keys live only on the server (the `/api/chat` handler); they are never sent to the browser.

## Seam

`createModelAdapter({ provider, model })` is the single place provider/model/key/base-URL are wired.
Swapping providers or pointing at a different gateway is a change here — nowhere else.
